Responsive Breakpoints

  • Breakpoints are viewport widths where layout / typography adapt to improve UX
  • Prefer content-driven breakpoints: change layout when design breaks, not to match devices
  • Mobile-first pattern: write base (small screens) then add @media (min-width: ...)
  • Common breakpoint examples (guideline, not rules):
    • small / phone: up to ~599px
    • tablet: 600–899px
    • small laptop: 900–1199px
    • desktop / large: 1200px+
  • Use few, meaningful breakpoints; prefer fluid techniques (flex, grid, clamp()) between breakpoints
  • Consider using em/rem for queries (scales with user font-size) or px for simplicity
