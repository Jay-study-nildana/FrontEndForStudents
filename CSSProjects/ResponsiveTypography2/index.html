Responsive typography that scales with intent

This demo shows techniques to keep text readable, balanced, and proportionate across view sizes using relative units, fluid scaling, and bounded sizes.

Maintain comfortable line-length and rhythm

Readability depends on line-length and consistent vertical rhythm. This container is bounded with a max measure (in ch) so lines stay between ~45–75 characters. Spacing uses a baseline rhythm variable so headings and paragraphs maintain consistent vertical spacing.

Fluid headings with sensible bounds

Headings below use clamp() to scale between a minimum and maximum value. The middle value uses viewport units so type grows smoothly between breakpoints without abrupt jumps.

kicker / small caps example

Small-caps can be useful for accent text but avoid tiny caps on small viewports; this demo disables small-caps at narrow widths to keep legibility.

Techniques & patterns

Relative units (rem / em)

Use rem for layout spacing and em for component-relative sizing. The example button below uses em so it scales with the parent's font-size.

Fluid scale (viewport-based)

clamp() with a vw-driven middle value gives smooth, bounded scaling across widths.

Accessibility: minimum sizes

We enforce minimum readable sizes with the lower bound of clamp(), and let users scale text with browser zoom. The demo control (JS) shows how the base font-size affects everything.
